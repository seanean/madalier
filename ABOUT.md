# About this project

Hi!

I work with data and I thought it would be useful to use AI to develop **a data modelling tool that makes my workflows easier**. The vision for this app was to create a **simple local app** with few dependencies that I can **use to create and manage versioned data models** and generate related files (DDLs, CSVs, diagrams).

In its current state, it works for me. If you would like to use it, feel free!

**I don't plan to continually add features**, but we will see as it seems every time I come back to it I have the thought _wouldn't it be nice if Madalier could do X?_

This project is also another **learning experiment about developing tools using AI**. Here are some methods I've tried:
1. (In this project) build the initial bones, schemas, logic of the project myself then iteratively implement each feature through prompting.
2. Plan to develop detailed requirements.md and plan.md files => implement => document in design.md => iterate while using cursor rules to (1) document bug fixes and feature requests, (2) store transcripts after successful implementation, and (3) keep project documentation up to date.
3. Plan to develop detailed plan.md file => implement => iterate without rules (and without doing much to document or capture changes, this was a precursor to #2.)
4. Get straight into agent prompting based on an initial idea, often for smaller scripts and tasks.

I have a lot to learn, so I will keep exploring ways I can use technology to make my life easier. Enjoy the project!
